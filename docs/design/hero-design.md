# Hero 섹션 설계

포트폴리오 첫 화면. 이름을 임팩트 있게 강조.

---

## 1. 현재 구현

### 핵심 인터랙션
- **커서 트레일**: 마우스를 움직이면 지나간 궤적에 이름이 드러남
- **커스텀 커서**: accent-warm 색 원형 커서 (섹션 내에서만 표시)

### 구성
| 요소 | 내용 |
|------|------|
| 이름 | `name` prop, `--color-accent-warm` 강조 |
| 디코이 단어 | 20여 개, `--color-text-muted` |
| 배경 | `--color-bg` |
| 마스크 | SVG mask + circle 트레일, objectBoundingBox 좌표계 |

### 성능
- 커서: ref + requestAnimationFrame으로 직접 DOM 갱신 (리렌더 없음)
- 트레일: requestAnimationFrame으로 화면 주사율에 맞춤 (초당 최대 60회 업데이트)
- 최대 100개 포인트(TRAIL_MAX), BRUSH_RADIUS: 0.05
- 브러시/커서 좌표: 컨텐츠 영역(contentRef) 기준으로 계산해 maxWidth 등 레이아웃과 일치

### 긁기 동작
- 마우스가 지나간 위치에 원형 마스크가 쌓임 → 그 영역의 텍스트가 드러남
- **사라짐 없음**: 한 번 드러난 영역은 그대로 유지됨 (트레일 포인트는 최대 100개로 FIFO)

### Props
```ts
type HeroProps = { name?: string };
```

## 2. "찾기" 컨셉 (적용됨)

### 아이디어
- 이름만 있으면 단조로움 → 여러 단어 속에서 **이름을 찾게** 만든다
- 이름은 **accent-warm** 색으로 구분, 나머지는 **text-muted**
- 커서 트레일로 드러나면서 "어디 있을까?" → "찾았다!" 순간을 만든다

### 구성
| 요소 | 처리 |
|------|------|
| 내 이름 (전체 한 단어) | `--color-accent-warm`, 강조 |
| 디코이 단어들 | `--color-text-muted`, 퍼블리셔/React/CSS 등 20여 개 |
| 배치 | 셔플 후 그리드, 이름 위치는 매 렌더마다 랜덤 |

### 원칙
- 찾기는 2~3초 안에 가능한 수준
- 디코이는 스킬/역할 관련으로 자연스럽게

## 4. 연동 UI (히어로 전용)

| 컴포넌트 | 설명 |
|----------|------|
| **HeroModal** | 히어로 로딩 후 3초 뒤 하단에서 슬라이드 업. 질문 "이 홈페이지를 제작한 사람은?", input 기본값 이름, 확인 시 Hero 숨기고 포트폴리오 본문으로 전환. 배경 어둡게 없음. |
| **Toast** | "화면을 긁어주세요" 안내. 배경 없이 보더(`--color-accent-warm`) + 그림자, 텍스트도 accent-warm. |

## 5. 기술 결정

- **Storybook 제외**: 마우스 인터랙션 중심이라 앱에서 직접 검증
- **sections/Hero**: feature가 아닌 섹션 단위로 `src/sections/`에 위치
- **HeroModal**: 히어로에서만 사용 → `shared/ui/HeroModal.tsx`로 네이밍
