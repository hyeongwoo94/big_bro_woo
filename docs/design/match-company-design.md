# MatchCompany 섹션 설계

Hero 다음 섹션. "내가 원하는 회사인가?"를 질문으로 확인하고, 모집자 입장에서 자기 회사와의 적합성을 체크하게 한다.

---

## 1. 개요

### 컨셉
- Hero에서 **이름을 각인**시켰다면, MatchCompany에서는 **나의 기준**을 제시한다.
- 모집자가 질문에 답하면서 "우리 회사가 맞는가?"를 스스로 확인한다.
- 합산 결과에 따라 **매치** 시 다음 섹션 진입, **미매치** 시 종료 흐름.

### 목적
- 일방적 구직이 아닌 **상호 선택** 톤 전달
- "나는 원하는 회사에만 지원한다"는 이미지
- 모집자 입장에서도 재미·기억에 남는 경험

---

## 2. UI/UX 흐름

### 2.1 인트로
- Q1 노출 전 **인트로 문구** 1회 표시
- 문구: **"솔직하게 대답해주세요"**
- 표시 후 (페이드 아웃 또는 짧은 유지) → Q1으로 전환

### 2.2 질문 단계
- 질문 1개씩 노출 (총 5문)
- **진행 표시**로 현재 위치 노출 (예: 1/5, 또는 점 5개)
- 각 질문마다 **2개의 답변 버튼** (기본 O/X, 텍스트 커스터마이징 가능)
- 답변 선택 시 → 다음 질문으로 전환
- 5번째 질문 답변 후 → 결과 단계로 전환

### 2.3 결과 단계

| 조건 | 화면 | 후속 동작 |
|------|------|-----------|
| **O 3개 이상** | "우리 잘 맞을 것 같아요" 등 긍정 메시지 | "다음 보기" 버튼 → 다음 섹션(Projects 등)으로 스크롤/전환 |
| **O 2개 이하** | 아래 미매치 메시지 | "확인" 버튼 → 미매치 시 동작 (3.4 참고) |

### 2.4 매치 기준 (확정)
- **매치**: O 개수 >= 3 → 다음 섹션 진입
- **미매치**: O 개수 < 3 → 미매치 화면 표시

---

## 3. 상세 요구사항

### 3.1 질문 구조
- **총 5문** 고정
- 각 질문: `{ id, text, yesLabel?, noLabel? }`
  - `yesLabel`: O에 해당하는 버튼 텍스트 (기본 `"O"` 또는 `"예"`)
  - `noLabel`: X에 해당하는 버튼 텍스트 (기본 `"X"` 또는 `"아니오"`)
- 텍스트는 설계/구현 시 별도 데이터로 관리 (예: `matchCompanyQuestions.ts`)

### 3.2 답변 버튼 (O/X 시각적 구분)
- 2개 버튼: 긍정(O) / 부정(X)
- **O(긍정)**: 액센트 강조 — `--color-accent` 또는 `--color-accent-warm` 배경/보더, 채워진 느낌
- **X(부정)**: 보조 톤 — 보더만 또는 `--color-text-muted`, 덜 강조
- 질문별로 `yesLabel`, `noLabel`로 **텍스트 커스터마이징 가능**
- 선택 시 시각적 피드백 (체크, 강조 유지 등)

### 3.3 결과 화면
- **매치**: 축하/긍정 메시지 + "다음 보기" → 다음 섹션으로 이동
- **미매치**: 아래 고정 메시지 + "확인" 버튼

**미매치 시 노출 문구**
> 방문해 주셔서 감사합니다. 귀하는 김개발과 함께 하실 수 없습니다.

### 3.4 미매치 시 동작 (확인 버튼 클릭 후)
"미매치 시 동작"은 사용자가 **"확인" 버튼을 눌렀을 때 기술적으로 무엇이 일어나는지**를 말한다.

- **1단계**: `window.close()` 호출 시도
- **제한사항**: 사용자가 직접 연 탭(주소창 입력, 북마크 등)은 브라우저 보안상 `close`가 차단됨. `window.open()`으로 연 팝업만 닫힘.
- **2단계**: close가 안 되면 화면에 "제가 원하는 회사가 아닙니다" 안내 노출 하고 아래에 확인버튼 추가하고 누르면 빈페이지가 보이게해줘.

## 4. 디자인

### 4.1 레이아웃
- **컨텐츠 라인**: `max-width: 1280px` (Hero 섹션과 동일)
- 콘텐츠는 가운데 정렬

### 4.2 진행 표시
- **위치**: 질문 영역 상단 또는 질문 바로 위
- **형식**: `1/5`, `2/5`, … `5/5` 또는 점 5개 중 현재만 강조 (● ○ ○ ○ ○)
- 질문 전환 시 진행 표시만 갱신 (애니메이션은 선택)

### 4.3 인트로
- **텍스트**: "솔직하게 대답해주세요"
- 인트로 노출 후 페이드 아웃(또는 짧은 유지) → Q1 진입

### 4.4 질문 애니메이션
- **첫 등장(페이드 인)**: 원근감 — 멀리에서 앞으로 다가오는 느낌
  - `scale(0.8)` + `opacity: 0` → `scale(1)` + `opacity: 1` 또는 `perspective` + `translateZ`
- **질문 → 다음 질문 전환** (가로 슬라이드 대신 참신한 연출):
  - **권장: 블러 포커스 전환** — 이전 질문이 `filter: blur()` + `opacity` 감소로 흐려지며 사라지고, 다음 질문은 흐린 상태에서 선명해지며 등장. 시선이 옮겨가는 느낌.
  - **대안 A: 줌 스루** — 이전 질문이 중앙으로 축소되며 사라지고, 다음 질문이 중앙에서 확대되며 등장 (터널 통과 느낌).
  - **대안 B: 카드 플립** — 질문 영역이 3D Y축 플립으로 뒤집히며 다음 질문 노출 (카드 뒤집기).

### 4.5 답변 버튼 타이밍·애니메이션
- 질문이 **페이드 인으로 다 나타난 후 1초** 대기
- 그 다음 **아래에서 위로** 답변 버튼(O/X) 슬라이드 업
  - `translateY(100%)` → `translateY(0)` 또는 `translateY(20px)` → `0`, 필요 시 `opacity` 병행
- O/X 버튼은 3.2(답변 버튼 시각적 구분)에 따라 스타일 유지

### 4.6 결과 화면 전환
- 5문 답변 완료 후 **0.3~0.5초** 짧은 대기 (결과 “계산” 느낌)
- 이어서 매치/미매치 메시지를 **원근감 페이드 인**으로 노출
- 결과용 버튼("다음 보기", "확인")은 **아래→위 슬라이드 업**

## 5. 데이터 구조 (안)

```ts
type MatchQuestion = {
  id: string;
  text: string;
  yesLabel?: string;  // 기본 '예'
  noLabel?: string;   // 기본 '아니오'
};

type MatchResult = 'match' | 'no-match';
// match: O >= 3
// no-match: O < 3
```

| 2 | 코드 리뷰 문화가 있나요? | 예 | 아니오 |
| 3 | 원격 근무·유연 근무를 지원하나요? | 예 | 아니오 |
| 4 | 성과보다 과정도 평가해 주시나요? | 예 | 아니오 |
