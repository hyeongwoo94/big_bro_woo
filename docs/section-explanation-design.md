# 섹션 설명(버튼 + 모달) 설계

각 섹션별로 "이 섹션을 어떤 기술·방식으로 만들었는지" 설명을 보여주는 재사용 UI. 퍼블→프론트 전향 포트폴리오에서 구현 스토리를 강조하기 위함.

---

## 1. 구성 요소

| 요소 | 역할 |
|------|------|
| **설명 버튼** | 클릭 시 해당 섹션의 설명 모달을 연다. |
| **설명 모달** | 제목 + 본문으로 기술/구현 내용을 표시. 닫기 시 배경으로 복귀. |

- 두 요소는 **세트로 재사용**: 여러 섹션에서 같은 컴포넌트를 쓰고, **내용(title, body)만 섹션별로 전달**.

---

## 2. 요구사항 (확정)

### 2.1 버튼 디자인
- **다른 공통 버튼과 구분**: "기술 설명 보기"용이라는 게 한눈에 들어오도록 별도 스타일(아이콘, 크기, 색 등) 적용. 공통 Primary/Secondary 버튼과 다르게 디자인.

### 2.2 반응형
- **768px 이하**: 설명 버튼과 모달 **둘 다 비표시** (모바일에서는 숨김).
- **769px 이상**: 버튼 표시, 모달도 정상 동작.
- 구현: CSS 미디어 쿼리 `min-width: 769px` (또는 768.01px)로 제어.

### 2.3 모달 배경
- 모달이 열릴 때 **뒷배경 어둡게** (오버레이). HeroModal과 구분되는 점.

### 2.4 모달 크기·애니메이션
- **최대 너비**: 1020px.
- **애니메이션**: 페이드 인/아웃 (opacity 또는 opacity + 약간의 scale 등으로 구현).

### 2.5 한 번에 하나만 열림
- 여러 섹션에 설명 버튼이 있어도 **동시에 하나의 설명 모달만** 열림. 다른 버튼을 누르면 기존 모달을 닫고 새 모달을 열거나, 전역에서 “열린 모달 하나”만 유지.

### 2.6 모달 위치
- 모달은 **브라우저(뷰포트) 가운데**에 배치. 세로·가로 모두 중앙 정렬.

---

## 3. 구현 설명에 들어갈 내용 (가이드)

이 포트폴리오는 **바이브코딩으로 제작**하고 있으며, 단순 구현에서 그치지 않고 **학습까지 이어지게** 기록하는 것을 목표로 한다. 각 섹션의 설명 모달에는 아래를 반영한다.

### 3.1 취지
- 이 포폴이 바이브코딩(AI와의 협업)으로 만들어지고 있다는 점을 짧게 언급.
- 구현을 넘어 **학습·이해한 내용**이 담기도록 작성 (나중에 다시 읽었을 때 복기 가능한 수준).

### 3.2 활용 → 결과
- **어떤 점을 활용하여 어떤 것을 만들었는지**를 구체적으로 적는다.
  - 예: "SVG mask의 objectBoundingBox 좌표계를 활용해, 마우스 궤적에 원형 브러시를 쌓아 단어가 드러나는 효과를 만들었다."
- 기술 스택·훅·API 이름만 나열하기보다, "무엇에 쓰였고 → 어떤 결과가 나왔는지" 한 문장씩 연결.

### 3.3 재활용 가능 시
- 해당 UI/컴포넌트를 **다른 개발자가 가져다 쓸 수 있도록** 사용 방법을 부가적으로 적는다.
  - 예: "이 컴포넌트를 쓰려면 App을 `TechNoteProvider`로 감싼 뒤, 섹션 안에 `<TechNote title="..." content={...} />`를 넣으면 된다."
- 파일 경로, 필요한 props, 주의사항(예: 768px 이하에서는 비표시) 등 포함하면 좋음.

---

## 4. 설계 시 추가로 정할 것

### 4.1 버튼
- **위치**: 섹션 내부 어디에 둘지 (예: 섹션 우측 상단, 헤더 옆, 우측 하단 고정 등).
- **표시 방식**: 아이콘만 / 텍스트만 / 아이콘+텍스트. 예: `?`, `i`, "기술 설명" 등.
- **접근성**: `aria-label`로 용도 설명 (예: "이 섹션 구현 설명 보기").

### 4.2 모달
- **닫기 방식**: 오버레이 클릭, ESC 키, 닫기(X) 버튼 — 셋 다 지원할지, 일부만 할지.
- **포커스**: 열릴 때 모달 안으로 포커스 가두기, 닫을 때 버튼으로 포커스 복귀 (a11y).
- **본문 형식**: 단순 텍스트만 / 줄바꿈·리스트 허용 / 마크다운 등 — 확정 시 컴포넌트 API(title, children 또는 content) 결정에 반영.

### 4.3 컴포넌트 분리
- **채택 이름**: `TechNote`(버튼/트리거), `TechNoteProvider`(상위 래퍼), 내부 모달은 `TechNoteModal`. 파일: `src/shared/ui/TechNote.tsx`.
- **데이터**: 각 섹션에서 `title`, `content`(또는 `children`)를 prop으로 넘기는 형태로 할지 결정.

---

## 5. 기술·위치 (구현)

- **구성**: `shared/ui/` 에 재사용 컴포넌트 (버튼 + 모달 또는 하나의 래퍼에서 둘 다 제공).
- **스타일**: 디자인 토큰(`--color-*`, `--spacing-*` 등) 사용, 768px 브레이크포인트 일관 적용.
